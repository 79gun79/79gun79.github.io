## **<span class="purplepen">HTTP</span>란?!**

- **H**yper **T**ext **T**ransfer **P**rotocol
- 즉, <span class="yellow2pen">종단 시스템간 데이터를 주고 받기 위한 통신 규약 또는 프로토콜</span>을 의미한다.  
- 초기에는 뜻에서 알 수 있듯이, <span class="bluepen">하이퍼텍스트 문서</span>를 전송하는 규약으로 동작했다.   
- 현재는 <span class="bluepen">거의 모든 형태의 데이터</span> 전송이 가능하다.
  - 예: `HTML`, `Text`, `Image`, `음성`, `영상`, `JSON`, `XML` 등... 

<br>

## **<span class="purplepen">HTTP</span> 특징**

### **1. 클라이언트 서버 구조**

- **Request Response 구조**
  - **<span class="bluepen">클라이언트</span>** : 서버에 **요청**을 보내고, 응답 대기
  - **서버** : 요청에 대한 결과를 만들어서 **응답**
- 해당 구조는 클라이언트와 서버로 개념적으로 분리함.
- 따라서, <span class="green2pen">각각의 개별적(특징적)인 진화</span>가 가능함.
  - 예: 클라이언트는 UI/UX, 서버는 비즈니스 로직에 집중

<br>

### **2. 무상태(Stateless) 프로토콜**

- 서버가 <span class="redpen">클라이언트의 상태를 보존하지 않음.</span>
- 장점: 서버 확장성이 높음
- 단점: 클라이언트가 데이터를 매번 추가적으로 전송해야함.

#### **실무에서는..**

- **무상태(Stateless)**는 서버 관리 측면에서 좋으나, **<span class="redpen">데이터를 많이 보내야 함</span>**.   
- **상태유지(Stateful)**는 최소한 사용, 그 외는 **무상태(Stateless)**로 설계.   

**무상태(Stateless)** : 단순 서비스 소개 화면  

**상태유지(Stateful)** : 로그인 화면

## **<span class="purplepen">HTTP/1.0</span> vs <span class="purplepen">HTTP/1.1</span>**   

### **HTTP 역사**

- HTTP/0.9 [1991]
- <span class="purplepen">HTTP/1.0</span> [1996]
- **<span class="purplepen">HTTP/1.1</span>** [1997]    
  <span class="redpen">가장 많이 사용하는 중요한 버전!</span>    
    - RFC2068 [1997]   
    - RFC2616 [1999] : 대부분의 공식문서 내용이   
    - RFC7230~7235 [2014] : 현재의 내용은 해당 버전    
- <span class="blue2pen">HTTP/2</span> [2015] → 성능 개선  
- <span class="blue2pen">HTTP/3</span> [~현재] → 성능 개선   

> 역사에서 알 수 있듯이, **HTTP**는 **1.1 버전**에서 비약적인 발전을 이뤄냈다.

### **두 버전의 가장 큰 차이**  

`1.0`과 `1.1` 버전의 큰 차이는 **지속 연결**이다!
