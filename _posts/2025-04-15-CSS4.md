## **전통적인 방식의 스타일링**

### **인라인 스타일**

- **HTML** 태그의 `style` 속성을 사용해서 CSS 스타일을 지정하는 방식
    - `style` 속성에는 **객체** 형식의 값이 할당.
    - `font-size`와 같은 케밥 케이스는 **카멜 케이스**로 작성.
    
    ```tsx
    export default function Example1() {
      return (
        <>
          <h1 style={{ color: "red", textDecoration: "underline" }}>내 웹사이트</h1>
        </>
      );
    }
    
    export default function Example2() {
      const style = { color: "red", textDecoration: "underline" };
      return (
        <>
          <h1 style={style}>내 웹사이트</h1>
        </>
      );
    }
    ```

### **글로벌(외부) 스타일**

- 별도의 **CSS 파일**에 CSS 코드를 작성하고, 컴포넌트 파일과 연결해서 사용하는 방법
- **css 파일** 생성 후, 코드 작성
    
    ```css
    .title {
      color: blue;
      text-decoration: underline;
    }
    <!-- App.css -->
    ```
    
- 컴포넌트 파일
    
    ```tsx
    import "./App.css";
    
    export default function App() {
      return (
        <>
          <h1 className="title">Test</h1>
        </>
      );
    }
    ```
    
- <span class="bluepen">컴포넌트 트리</span>에 연결된 다른 컴포넌트 파일     
  → 한번 더 <span class="pinkpen">import</span>를 하지 않아도, **css**가 **전역으로 적용**되어 사용 가능.    
    
    ```tsx
    export default function 다른컴포넌트() {
      return (
        <>
          <h1 className="title">Test</h1>
        </>
      );
    }
    ```
    
- 좀 더 용이하게 관리하려면, `css` 폴더를 하나 생성하여 **css 파일을 관리**하여 불러오자.

### **CSS 모듈**

- **글로벌 스타일 방식**은 <u>컴포넌트 트리 내의 모든 컴포넌트가 스타일이 적용</u>된다는 특징이 있다.   
- 이 방식은 **<span class="redpen">모듈 방식</span>**을 사용해서 <span clas="yellow2pen">**특정 컴포넌트**에만 종속되는 CSS 코드를 작성</span>할 수 있다.    
- 외부 스타일 방법과 비슷하지만, 파일명이 **`*.module.css`**라는 특징이 있다.    
- **css 파일** 생성 후, 코드 작성 → 이 때, 파일명은  `*.module.css`로 끝나야 한다.
    
    ```css
    .underline {
      text-decoration: underline;
    }
    <!-- Footer.module.css -->
    ```
    
- 특정 컴포넌트에 **특정 css** 적용.
    
    ```tsx
    import **styles** from "./Footer.module.css";
    
    export default function Example() {
      return (
        <>
          <footer>
            <p className={styles.underline}>&copy; 2024 내 웹사이트. 모든 권리 보유.</p>
          </footer>
        </>
      );
    }
    ```   
       
    다음과 같이 **<span class="redpen">모듈 방식</span>**을 사용하면, <span class="red2pen">클래스명이 암호화</span>되어 나타나는 것을 확인 가능.       
    
    <img width="407" height="46" alt="Image" src="https://github.com/user-attachments/assets/5ba6b93e-17b6-4389-9f38-8b793f537f6c" />   
    
- 특정 컴포넌트에 **여러 css** 적용.    
  **`**(백틱)을 사용하여, 공백으로 구분하여 나타낸다.   
    
    ```tsx
    import styles from "./Footer.module.css";
    
    export default function Example() {
      return (
        <>
          <footer>
            <p className={`${styles.underline} ${styles["red-color"]}`}>
              &copy; 2024 내 웹사이트. 모든 권리 보유.
            </p>
          </footer>
        </>
      );
    }
    ```

### **<span class="orangepen">classnames 패키지</span> 사용**

1. 먼저, **<span class="orangepen">classnames 패키지</span>** 설치    
    
    ```bash   
    npm i classnames
    ```  
    <img width="354" height="169" alt="Image" src="https://github.com/user-attachments/assets/4a4d4987-1338-4bee-b03c-c660cdb8ac55" />   
       
     
2. 이전에 작성한 **<span class="redpen">CSS 모듈 예제</span>**를 가져와 다음과 같이, 수정한다.     
    
    ```tsx
    import styles from "./Footer.module.css";
    import classNames from "classnames/bind"; // classnames 패키지 불러오기
    
    export default function Example() {
      const cx = classNames.bind(styles);  // 특정 css 바인딩
      return (
        <>
          <footer>
            <p className={cx("red-color", "underline")}>
              &copy; 2024 내 웹사이트. 모든 권리 보유.
            </p>
          </footer>
        </>
      );
    }
    ```     

    - `"classnames/bind"`: **<span class="orangepen">classnames 패키지</span>**의 bind 불러오기.      
    - `classNames.bind(CSS명)`: 특정 css 바인딩.      
    - **`패키지바인딩명(속성값나열)`**: 콤마(,)로 구분하여 여러개 적용 가능.     
     
- 자세한 사용법은 npm 패키지에 명시되어 있음.
