## **<span class="purplepen">HTTP</span>**와 **<span class="purplepen">HTTP</span>**의 차이는 무엇인가?   

- **<span class="purplepen">HTTP</span>**는 <span class="yellow2pen">종단 시스템간 데이터를 주고 받기 위한 통신 규약 또는 프로토콜</span>을 의미한다.      
- 그렇다면 **<span class="purplepen">HTTPS</span>**는 무엇인가?      

---

### **<span class="purplepen">HTTP</span>**

- **HTTP**: **H**yper **T**ext **T**ransfer **P**rotocol
- 초기, 하이퍼텍스트 문서를 전송하는 규약   
- 현재, <span class="blue2pen">거의 모든 형태의 데이터</span> 전송 가능       
  HTML, Text, Image, 음성, 영상, 파일, JSON, XML 등..
- 서버간에 데이터 송수신 → 대부분, HTTP로   

### **HTTP 역사**

- HTTP/0.9 [1991]  
- HTTP/1.0 [1996]   
- **<span class="redpen">HTTP/1.1</span>** [1997]     
  <span class="yellow2pen">가장 많이 사용하는 중요한 버전!</span>    
    - RFC2068 [1997]   
    - RFC2616 [1999] : 대부분의 공식문서 내용이 포함됨   
    - RFC7230~7235 [2014] : 현재의 내용은 해당 버전   
- HTTP/2 [2015] → 성능 개선   
- HTTP/3 [~현재] → 성능 개선   

### HTTP 기반 프로토콜

- TCP: **<span class="redpen">HTTP/1.1</span>**, HTTP/2
- UDP: HTTP/3     
  현재는 **HTTP/1.1**을 주로 사용하고, HTTP/2, HTTP/3도 점점 증가하는 추세  
  [Network] - [Protocol 내용 확인]       
     
  <img width="414" height="314" alt="Image" src="https://github.com/user-attachments/assets/c8f7c414-4775-43e6-9f45-3ec2e8cb13ce" />      

> **HTTP**는 OSI 네트워크 모델의 **애플리케이션 계층 프로토콜**이다.  
  
<br>

---

### **<span class="purplepen">HTTPS</span>**

- **HTTP** + <span class="bluepen">Secure</span>
- 즉, HTTP에 <span class="bluepen">데이터 암호화</span>가 추가된 버전이라고 생각하면 된다.   
- 데이터 전송 보호를 위한 방법으로 **SSL**과 **TLS**가 있다.    

### **SSL / TLS** 암호화 기술

- **S**ecure **S**ockets **L**ayer     
- **T**ransport **L**ayer **S**ecurity    
- 도청되어도 내용을 알 수 없도록 <span class="blue2pen">암호화</span>시키거나,   
- 올바른 대상인지 확인하는 <span class="blue2pen">인증절차</span>를 거치게 한다.  

> 따라서, **HTTPS**는 보안이 중요한 서비스에 사용되면서, 일반적인 구조로 자리잡았다.
